Getting Started

ironyLabs Links Memex is a personal link collection displayed as an interactive mind map. Access it at:

URLPurpose
http://localhost:8098Public links page (mind map / pinboard / tag cloud)
http://localhost:8098/admin.htmlAdmin panel (manage links)
http://localhost:8098/screensaver.htmlScreensaver mode
http://localhost:8098/api/rssRSS/Atom feed

Note: Replace 8098 with your custom port if you changed APP_PORT in .env.

Configuration (.env file)

All settings are controlled from a single .env file. Copy from the template and edit:

cp .env.example .env
nano .env
VariableDefaultDescription
ADMIN_PASSWORDironyLabs2024!Admin login password
SECRET_KEYchange_this...JWT signing key (use a long random string)
APP_PORT8098Port the app runs on

That's it. No other files need editing. Run docker compose up -d --build and the app uses your settings.

Changing the Port

To run on a different port (e.g. 9000), just edit .env:

APP_PORT=9000

Then restart: docker compose down && docker compose up -d

The port change applies to both the Docker gateway and the internal nginx listener automatically. No need to edit docker-compose.yml or nginx config files.

Android app: Not affected — the app asks you to enter the full URL including port (e.g. http://yourserver:9000), so it works with any port.

Search

Public

Press Ctrl+K or / to open the search overlay. On mobile, tap the 🔍 button (top-right).

Search matches against: title, category, description, URL, and tags. Use arrow keys to navigate results, Enter to open.

Navigate to /admin.html. Login with:

Android Companion App

Mobile

The Memex Share Android app lets you save links from any app directly to your server using Android's share sheet.

Installation

  1. Transfer Memex-Share-debug.apk to your Android device
  2. Open it and allow "Install from unknown sources" when prompted
  3. The app appears as "Memex Share" in your app drawer

Setup

  1. Open the app
  2. Enter your server URL (e.g. http://192.168.1.100:8098 or https://yourdomain.com)
  3. Enter your admin password
  4. Tap "Connect" — you should see "✓ Connected"

Sharing Links

  1. Browse any page in Chrome, YouTube, Twitter, etc.
  2. Tap the Share button (in any app)
  3. Select "Save to ironyLabs" from the share sheet
  4. Edit the title/category if needed
  5. Tap "Save" — link is instantly added to your server

Building the APK from Source

The Android source code is in the android/ folder of the project. You can build it yourself if you want to modify the app or don't trust pre-built APKs.

Requirements

  • Java JDK 17+ (OpenJDK recommended)
  • Android SDK (install via Android Studio or command-line tools)
  • Android SDK Platform 35 (install via SDK Manager)
  • Android Build Tools 35.0.1+

Option A: Command Line (no Android Studio)

# Set ANDROID_HOME to your SDK location
# Windows:
set ANDROID_HOME=%LOCALAPPDATA%\Android\Sdk

# Navigate to the android folder
cd android

# Build debug APK
gradlew.bat assembleDebug

# APK output location:
# android/app/build/outputs/apk/debug/app-debug.apk

Option B: Android Studio

  1. Open Android Studio
  2. File → Open → select the android/ folder
  3. Wait for Gradle sync to complete
  4. Build → Build Bundle(s) / APK(s) → Build APK(s)
  5. APK appears in android/app/build/outputs/apk/debug/

Installing the APK

# Via ADB (phone connected via USB with USB debugging enabled):
adb install app/build/outputs/apk/debug/app-debug.apk

# Or transfer the APK to your phone and open it

Standalone Mode (No Docker)

Run the full application without Docker — just Python. Status: Experimental. Docker Compose is the recommended and fully tested deployment method.

Requirements

  • Python 3.10, 3.11, or 3.12 (recommended)
  • No Docker needed
  • pip (comes with Python)

Quick Start (Windows)

# Double-click start-standalone.bat
# OR manually:
cd standalone
pip install -r requirements.txt
python run.py

The app starts at http://127.0.0.1:8098 and opens your browser automatically.

Quick Start (Linux/Mac)

cd standalone
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt
python run.py

How It Works

  • Same backend code as Docker version (main.py)
  • SQLite database stored in standalone/data/links.db
  • Uploads stored in standalone/data/uploads/
  • Frontend served by FastAPI directly (no Nginx needed)
  • Single process, single port
  • Auto-opens browser on startup

Known Issues & Python Compatibility

IssueCauseFix
Python 3.13+ fails to install dependenciespydantic-core requires Rust compilation on 3.13 and may fail if Rust toolchain isn't installedUse Python 3.11 or 3.12 instead. These have pre-built binary wheels for all dependencies
Build error mentioning "PyO3" or "maturin"Same as above — pydantic-core can't find compatible wheel for your Python versionDowngrade to Python 3.11 or 3.12
Module not found errorsRunning python run.py from wrong directoryMust run from the project root (not from inside standalone/)
Port already in useDocker instance or another app on port 8098Stop Docker first (docker compose down) or set PORT=9000 env var

Testing Standalone Mode

  1. Ensure Docker is stopped: docker compose down (frees port 8098)
  2. Run start-standalone.bat (Windows) or the manual steps above
  3. Browser should open to http://127.0.0.1:8098
  4. Login with the default password (set via ADMIN_PASSWORD env var, defaults to ironyLabs2024!)
  5. Add a link, verify it appears on the public page
  6. Press Ctrl+C in the terminal to stop

Sharing Links (Private Share URLs)

Admin · Recipients: No login needed

Create private, read-only share links that let anyone view a subset of your links without an account.

Creating a Share

  1. Open admin panel → click 🔗 Shares
  2. Enter a name (e.g. "My Music Links")
  3. Choose scope:
    • All Public Links — shares everything that's active + non-private
    • Category — shares only links in a specific category (enter category name in target field)
    • Collection — shares only links in a specific collection (enter collection slug)
  4. Click Create Share
  5. Copy the generated URL and send it to your friend

Share URL Format

https://yoursite.com/shared.html?token=ESbiZ0029Lm11mzY24Uy16MU6UjlLV2oNBwca0c52UY

Anyone who opens this URL sees a read-only card grid of the shared links. No login, no account needed.

Managing Shares

ActionHow
View all sharesAdmin → 🔗 Shares
Copy share URLClick the URL field (auto-copies)
Revoke accessClick "Revoke" — link stops working immediately
Create scoped shareSet type to "Category" or "Collection" + enter target name

What Recipients See

  • A dark-themed card grid showing link icons, titles, descriptions, and categories
  • Click any card to open the link in a new tab
  • The page shows the owner name, share name, and link count
  • No admin access, no ability to edit or delete anything

Security

  • Share tokens are 32-character cryptographically random strings
  • Tokens are unguessable — only people you give the URL to can access it
  • Private links are never included in shares (double-filtered server-side)
  • Revoking a share immediately removes access — no caching

Friends (Federated Read-Only Browsing)

Admin

Connect to friends who also run ironyLabs and browse their shared links directly from your instance.

Prerequisites

  • Your friend must have their own ironyLabs instance running (on their VPS)
  • Your friend must create a share and give you: their server URL + share token

Adding a Friend

  1. Open admin panel → click 👥 Friends
  2. Enter their display name (e.g. "Alice")
  3. Enter their instance URL (e.g. https://alice-links.com)
  4. Enter the share token they gave you
  5. Click Add Friend
  6. Click Test to verify the connection works

How It Works (Technical)

Your browser → Your server: GET /api/friends/1/links
Your server → Friend's server: GET /api/shared/{token}
Friend's server → Your server: { owner: "Alice", links: [...] }
Your server → Your browser: renders friend's links

Your server acts as a proxy — this avoids CORS issues and keeps the friend's token private (never exposed to the browser).

Browsing Friend's Links

Access MethodHow
Via APIGET /api/friends/{id}/links — returns friend's shared links as JSON
Via Admin"Test" button shows link count from friend
Via Public PageFuture: friend nodes on mind map (planned enhancement)

Example: Full Flow Between Two People

StepWhoAction
1YouCreate a share → get token abc123
2YouSend friend: "My URL is https://mysite.com, token is abc123"
3FriendIn their admin → 👥 Friends → adds your URL + token
4FriendClicks "Test" → sees "✓ Got 15 links from ironyLabs"
5FriendCreates their own share → gives you their token
6YouAdd them as a friend with their URL + token
7BothCan now browse each other's shared links (read-only)

Important Notes

  • Read-only: Friends can only view your links — never edit, delete, or add
  • Live data: Links are fetched fresh each time — not cached or stored
  • Revocable: Delete your share token and the friend loses access instantly
  • Private links: Never exposed through shares, regardless of scope
  • No account needed: The share token is the only credential
  • Self-hosted: Data stays on each person's server — no central service

Testing Locally (Docker Networking)

When testing the friends feature on the same machine (adding yourself as a friend), you cannot use localhost:8098 as the instance URL. The backend runs inside a Docker container, and localhost inside that container refers to itself — not your host machine.

Use the Docker internal hostname instead:

http://ironylabs_gateway:8098

This is the container name on Docker's internal network. The backend can reach it because all containers share the same Docker Compose network.

ScenarioInstance URL to Use
Testing on same machine (self as friend)http://ironylabs_gateway:8098
Friend on same local networkhttp://192.168.x.x:8098 (their LAN IP)
Friend on the internet (production)https://friend-domain.com (their public URL)

In production (deployed on separate VPSes with domains), you use the actual public URL — this issue only affects local Docker testing.

Without Their Own Instance (Simple Viewer)

Friends who don't run Memex can still view your shared links — just send them the share URL directly:

https://yoursite.com/shared.html?token=YOUR_TOKEN

They'll see a simple, styled card grid of your shared links. No login, no install. Works on any device.

Connecting via Cloudflare Tunnel

If both instances are behind Cloudflare Tunnels (no public ports), the connection works transparently:

  1. Both instances exposed via tunnel (e.g. https://yourdomain.com, https://frienddomain.com)
  2. You create a share → send friend your domain + token
  3. Friend adds you in their 👥 Friends panel using your https:// URL
  4. Friend clicks Test → their backend fetches from your tunnel → works

Why it works: The friend fetch is server-to-server (your backend → HTTPS → Cloudflare → friend's VPS). It's a standard HTTPS GET request — Cloudflare Tunnel handles it like any web traffic. No ports to open, no CORS issues, no special configuration.

Verify from terminal:

curl https://frienddomain.com/api/shared/THEIR_TOKEN

If that returns JSON with links, the connection works.

Cloudflare WAF note: If Cloudflare's Web Application Firewall is set to strict mode, it might block API requests from servers. Check your Cloudflare dashboard → Security → WAF and ensure /api/* paths aren't being challenged.
